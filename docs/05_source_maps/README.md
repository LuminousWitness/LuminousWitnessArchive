# Source Maps

Source maps are bridge documents that turn a large evidence ecology into navigable rooms.

A source map does not replace the primary record. It helps readers understand where to enter, which files belong together, what claims each cluster supports, and what still needs verification.

## Current Maps

- [`luminous_witness_master_cartography.md`](luminous_witness_master_cartography.md) — first high-level structural blueprint for the archive ecology.

## Map Requirements

Each source map should include:

1. the source cluster or notebook it summarizes;
2. major rooms or thematic sections;
3. claims supported by the cluster;
4. evidence types present;
5. representative source paths;
6. missing receipts or uncertainty tags;
7. links back to the Claims Ledger.

## Scope Note

Source maps are interpretation layers. They should be useful, readable, and honest about uncertainty. They should not imply that the whole archive has been finally proven or completed.

> A map is not the forest. A map helps the witness walk without losing the path.
