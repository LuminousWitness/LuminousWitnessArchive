## Primary Records and Receipts

Primary transcripts, screenshots, exports, source files, public statements, metadata, and hashes should be indexed here as they are accessioned.

Recommended future subdivisions:

```text
primary-records/
  transcripts/
  screenshots/
  exports/
  public-statements/
  source-documents/

receipts/
  metadata/
  hashes/
  release-notes/
  archived-links/
  corroboration/
```

Each primary record should identify:

- title;
- date;
- participants;
- model/platform/version;
- source type;
- original filename;
- canonical repository path;
- human-readable mirror, if any;
- verification status;
- related claims and frameworks.

## Document Status Vocabulary

Use these terms consistently:

- **Primary record** — original source preserved with minimal alteration.
- **Canonical** — current official archive version.
- **Working draft** — incomplete or actively changing.
- **Framework** — proposed explanatory structure.
- **Synthesis** — interpretation integrating multiple sources.
- **Source map** — navigation and cluster interpretation layer.
- **Public-discourse record** — media, social, institutional, or legal material.
- **Ledger note** — dated archive observation or event record.
- **Deprecated** — retained for provenance but replaced by a newer canonical version.
- **Unverified** — source, date, path, or claim requires confirmation.

## Accession Rule

No canonical artifact should enter the archive without:

1. a stable path;
2. an index entry;
3. document type and status;
4. authorship or contributor information;
5. source/provenance note;
6. date added;
7. related claim or category;
8. clear distinction between primary evidence and interpretation.

Use the reusable template:

- [`docs/_templates/archive_entry_template.md`](docs/_templates/archive_entry_template.md)

## Maintenance Rule

When a file is renamed, moved, superseded, or corrected:

- update this index;
- preserve the old history in Git;
- add a migration or correction note when the change affects citations;
- avoid silently breaking links.
